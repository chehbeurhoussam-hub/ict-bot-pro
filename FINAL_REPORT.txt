6. DEPLOYMENT GUIDE
═══════════════════════════════════════════════════════════════════════════════

METHOD 1: Docker (Recommended)
────────────────────────────────
  $ cp .env.example .env
  $ # Edit .env with your API keys
  $ docker-compose up -d
  $ # Access dashboard at http://localhost:7860

METHOD 2: systemd (Linux Server)
──────────────────────────────────
  $ sudo cp ict-bot.service /etc/systemd/system/
  $ sudo systemctl daemon-reload
  $ sudo systemctl enable ict-bot
  $ sudo systemctl start ict-bot
  $ sudo journalctl -u ict-bot -f

METHOD 3: Manual
──────────────────
  $ python -m venv venv
  $ source venv/bin/activate
  $ pip install -r requirements.txt
  $ cp .env.example .env
  $ # Edit .env
  $ python app.py

9. QUICK START
═══════════════════════════════════════════════════════════════════════════════

STEP 1: Get Binance API Keys
  → https://www.binance.com/en/my/settings/api-management
  → Create API key with Futures permissions ONLY
  → Enable IP whitelist
  → DISABLE withdrawal permission

STEP 2: Create Telegram Bot (Optional)
  → Message @BotFather on Telegram
  → Create new bot, get token
  → Get your chat ID from @userinfobot

STEP 3: Configure Environment
  → Copy .env.example to .env
  → Fill in BINANCE_API_KEY, BINANCE_SECRET
  → Set USE_SANDBOX=true (for testing)
  → Add TELEGRAM_BOT_TOKEN and TELEGRAM_CHAT_ID

STEP 4: Test in Sandbox
  → Run: python app.py
  → Open http://localhost:7860
  → Click "Run Backtest" to verify strategy
  → Monitor for 1-2 days

STEP 5: Go Live (When Ready)
  → Set USE_SANDBOX=false
  → Start with small balance ($100-500)
  → Monitor closely for first week
